---
title: "HTTP APIs"
---

WalletKit provides a set of HTTP APIs so you can integrate with your own backend service in any language or framework.


#### API Hosts
WalletKit exposes two endpoints, one for each environment:

- Mainnet: https://mainnet.walletkit.com
- Testnet: https://testnet.walletkit.com

#### Authentication
WalletKit uses Basic Authentication to authenticate your requests. Broadly, this works by passing the
`Authorization: Basic <credentials>` header where `<credentials>` is a base64 encoded string: `Project ID:API Secret`.

<Tip>
  Grab your ProjectID and API Key from the API Keys page in the WalletKit Dashboard:
  https://app.walletkit.com/api-keys
</Tip>

#### Example Request
Here is an example request to create a wallet on the Base testnet network:

```bash
curl --request POST \
--url https://testnet.walletkit.com/wallets \
--header 'X-WalletKit-Project-ID: <Project ID>' \
--header 'Authorization: Bearer <API Key>' \
--data '{
  "network": "Base",
  "name": "My Wallet",
  "control_mode": "developer",
  "developer_secret": "testnet-secret"
}'
```

Checkout the API Reference to see all the available endpoints: https://docs.walletkit.com/api-reference
